Authentication

Two-Tier Authentication System

Our API uses a secure two-tier authentication system:

  1. API Key (long-lived, secret) → Request token
  2. Active Token (short-lived, disposable) → Make requests
Step 1: Request an Active Token

Use your API key to request an active token:

Request: 
POST {{ request.scheme }}://{{ request.get_host }}/api/v1/auth/token
Content-Type: application/json

{
    "api_key": "your-api-key-here",
    "lifetime_minutes": 60,      // Optional: 1-1440 (1 min to 24 hours)
    "max_uses": 100              // Optional: 1-10000 requests
}
Response: 
{
    "success": true,
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expires_at": "2025-10-20T11:00:00Z",
    "lifetime_minutes": 60,
    "max_uses": 100,
    "created_at": "2025-10-20T10:00:00Z"
}
Token Configuration:
  • lifetime_minutes: How long the token stays valid (default: 60 minutes, max: 1440 minutes / 24 hours)
  • max_uses: Maximum number of API calls with this token (default: 100, max: 10,000)
  • You can customize these values per token request
  • Tokens expire when time limit OR usage limit is reached (whichever comes first)
Step 2: Use the Token for API Calls

Include the token in the X-API-Token header for all data requests:

Request: 
GET {{ request.scheme }}://{{ request.get_host }}/api/v1/data/YieldData?page=1&page_size=10
X-API-Token: your-active-token-here
Why Two-Tier Authentication?
  • Enhanced Security: Your API key is only exposed when requesting a token (not on every request)
  • Limited Risk: If a token is stolen, it expires quickly and has limited uses
  • Better Control: Each token has its own lifetime and usage limits
  • Easy Revocation: Tokens expire automatically without revoking the API key
Security Best Practices:
  • Never share your API key - It's like a password
  • Use HTTPS only - Never send keys or tokens over HTTP
  • Store keys securely - Use environment variables, not code
  • Use active tokens for all requests - Don't send API key with every request
  • Request short-lived tokens - Use minimum lifetime needed for your task
  • Monitor token usage - Check for unauthorized access in logs
Token Expiration & Renewal
Token Expires When:
  • Time limit reached (e.g., 60 minutes)
  • OR Usage limit reached (e.g., 100 requests)
  • Whichever comes first
When Token Expires:
  • Request a new token with your API key
  • Your API key remains valid
  • No need to contact administrator

GET Get Table Data
GET /api/v1/data/{table_name}?page=1&page_size=100&order_by=id

Headers:

  • X-API-Token: your-active-token (Required - use token from Step 1)

Query Parameters:

  • page: Page number (default: 1, min: 1)
  • page_size: Records per page (default: 100, max: your table permission limit)
  • filter: JSON object with filter conditions (optional)
  • fields: Comma-separated list of fields to return (optional, default: all allowed)
  • order_by: Field name to order by (optional, prefix with - for descending)

Example with filters:

Advanced Filtering System

New Feature: Intelligent Filtering

The API now supports advanced filtering with automatic field type detection and multiple filter operators!

Date Range Filtering

Filter data by date ranges for DateTimeField and DateField columns:

Single Date 
{"day_date": "2024-01-15"}
Get records for specific date
Date Range 
{"day_date": {
    "start": "2024-01-01",
    "end": "2024-12-31"
}}
Get records within date range
Text Search Filtering

Search and filter text fields with multiple operators:

Exact Match 
{"asset_code": "KR_BW_19"}
Exact string match
Contains Search 
{"asset_name": {"contains": "KR_BW"}}
Case-sensitive contains
Case-Insensitive 
{"asset_name": {"icontains": "solar"}}
Case-insensitive contains
Starts With 
{"country": {"startswith": "KR"}}
Starts with specific text
Numeric Range Filtering

Filter numeric fields with range operators:

Exact Value 
{"daily_prod_rec": 150.5}
Exact numeric match
Min/Max Range 
{"daily_prod_rec": {
    "min": 100,
    "max": 1000
}}
Between min and max values
Greater Than 
{"daily_prod_rec": {"gte": 100}}
Greater than or equal
Less Than 
{"daily_prod_rec": {"lt": 1000}}
Less than
Combined Filtering

Combine multiple filters for complex queries:

Multiple Filters Example 
{
    "asset_code": {"contains": "KR_BW"},
    "day_date": {"start": "2024-01-01", "end": "2024-12-31"},
    "daily_prod_rec": {"gte": 100}
}
Get KR_BW assets from 2024 with production ≥ 100 kWh
Automatic Field Detection

The system automatically detects field types and applies the appropriate filtering:

  • DateTimeField/DateField → Date range filtering
  • CharField/TextField → Text search filtering
  • FloatField/IntegerField → Numeric range filtering
Usage Examples

GET /api/v1/data/YieldData?filter={"day_date": {"start": "2024-01-01", "end": "2024-12-31"}}

Get all yield data for 2024

GET /api/v1/data/AssetList?filter={"asset_name": {"icontains": "solar"}}

Find all assets with "solar" in the name (case-insensitive)

GET /api/v1/data/YieldData?filter={"daily_prod_rec": {"gte": 1000}}

Get records with daily production ≥ 1000 kWh

GET /api/v1/data/YieldData?filter={
    "asset_code": {"contains": "KR_BW"},
    "day_date": {"start": "2024-06-01"},
    "daily_prod_rec": {"min": 500, "max": 2000}
}

Get KR_BW assets from June 2024 onwards with production between 500-2000 kWh

Important Notes
  • Date Format: Use YYYY-MM-DD format (e.g., "2024-01-15")
  • JSON Format: Filter parameter must be valid JSON
  • Field Names: Use exact database field names (case-sensitive)
  • Performance: Combined filters are optimized for database performance

How It Works

Complete Authentication Flow
Step 1️⃣: Get Your API Key

Generate an API key from the My API Dashboard. Your API key is a long-lived secret (like a password) that identifies you.

Important: API keys are shown only once! Save it securely.
Step 2️⃣: Request an Active Token

Send your API key to /api/v1/auth/token to get a short-lived active token. 

POST /api/v1/auth/token
{
  "api_key": "pk_xxxxx",
  "lifetime_minutes": 60,
  "max_uses": 100
}
Token Properties:
  • Lifetime: 1-1440 minutes (default: 60 min = 1 hour)
  • Max Uses: 1-10,000 requests (default: 100)
  • Expiration: Whichever limit is reached first
Step 3️⃣: Use Token for Data Requests

Include the token in the X-API-Token header for all data and aggregate requests. 

GET /api/v1/data/YieldData
X-API-Token: your-active-token-here
Why Use Tokens? Your API key is only exposed once (when requesting the token), not on every request. This is much more secure!
Step 4️⃣: Token Expires? Request a New One

When your token expires (after 60 minutes or 100 uses), simply request a new token with your API key. Repeat Step 2!

Code Examples

import requests

# Step 1: Get active token
api_key = "your-api-key-here"
token_response = requests.post(
    "{{ request.scheme }}://{{ request.get_host }}/api/v1/auth/token",
    json={
        "api_key": api_key,
        "lifetime_minutes": 60,
        "max_uses": 100
    }
)
token = token_response.json()["token"]

# Step 2: Fetch data with advanced filtering
headers = {"X-API-Token": token}
response = requests.get(
    "{{ request.scheme }}://{{ request.get_host }}/api/v1/data/YieldData",
    headers=headers,
    params={
        "page": 1, 
        "page_size": 10,
        "filter": '{"day_date": {"start": "2024-01-01", "end": "2024-12-31"}, "daily_prod_rec": {"gte": 100}}'
    }
)
data = response.json()
print(data)
