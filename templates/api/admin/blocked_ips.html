About Brute Force Protection
How It Works

The API brute force protection system automatically monitors and blocks malicious authentication attempts:

  • Failed Attempt Tracking: Every invalid API key attempt is recorded with IP, timestamp, and reason
  • Automatic Blocking: IPs exceeding {{ config.max_attempts }} failed attempts within {{ config.window_minutes }} minutes are automatically blocked
  • Progressive Delays: Exponential backoff delays are applied to slow down attackers (1s → 2s → 4s → 8s → 16s)
  • Rate Limiting: Token request endpoint is rate-limited to {{ config.token_rate_limit }} requests per minute per IP
  • Integration: Severe attacks (5+ failed attempts) trigger the main security system's permanent block
Unblocking IPs

Blocks expire automatically after {{ config.block_duration }} minutes. You can also manually unblock IPs using the "Unblock" button above. If a legitimate user reports being blocked, verify their identity before unblocking.

Adjusting Protection Levels

To modify protection thresholds, edit api/brute_force_protection.py and adjust the configuration constants:

  • MAX_ATTEMPTS_PER_IP - Number of failed attempts before blocking
  • ATTEMPT_WINDOW_MINUTES - Time window for counting attempts
  • BLOCK_DURATION_MINUTES - How long to block IPs
  • TOKEN_REQUEST_LIMIT - Rate limit per minute
