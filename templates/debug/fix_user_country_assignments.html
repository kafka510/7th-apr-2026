How This Works
Problem:

When you select countries in the user management interface, the system sometimes assigns individual AssetList records instead of proper country assignments. This causes:

  • Display issues showing all asset codes instead of just countries
  • Future sites in those countries won't be automatically accessible
  • Inefficient database queries
Solution:

This tool fixes the assignments by:

  • Identifying distinct countries from the assigned AssetList records
  • Clearing the current assignments
  • Assigning one representative AssetList record per country
  • Doing the same for portfolios
Example:

Instead of having 50 individual AssetList records assigned (showing as "SG, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, JP, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, KR, TW"), it will show just the distinct countries: "SG, KR, JP, TW".
